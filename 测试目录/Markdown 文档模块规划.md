# Markdown 文档模块规划

## 规划目标

Markdown 文档模块用于统一管理外部文档仓库的拉取、同步、目录入库和内容解析流程。整体以后端处理为主，前端只负责展示目录、渲染 HTML 和标题跳转。

## 整体流程

1. 在环境变量中配置一个目录，用于按仓库独立存放从 GitHub 拉取的 Markdown 文档仓库。
2. 在 `src/utils` 中封装常用 Git 脚本，统一处理克隆、拉取、推送、状态检查等操作。
3. `document` 模块提供文档仓库同步服务，负责调用 Git 脚本完成仓库更新。
4. 新仓库首次接入时，全量扫描仓库目录和文件结构，并将目录信息保存到数据库。
5. 后续仓库拉取更新后，基于文件变化重新比对目录结构，同步更新数据库记录。
6. 当前端请求指定 Markdown 文件时，后端读取文件内容，解析为 HTML，并提取标题目录后返回给前端。

## 模块规划

### 1. 文档仓库存放

- 在环境变量中配置的目录下集中存放外部文档仓库。

- 每个文档仓库独立一个子目录，避免相互影响。

- 目录本身不纳入版本管理，只保留仓库元信息和数据库索引数据。

### 2. Git 工具脚本

- 在 `src/utils` 下封装通用 Git 脚本。

- 统一对外提供基础能力，例如克隆仓库、拉取更新、推送变更、获取当前状态和最近提交信息。

- 后续 `document` 模块或其他模块有 Git 相关需求时，优先复用该工具层。

### 3. 文档同步服务

- `document` 模块增加仓库同步服务，负责管理文档仓库的拉取和推送流程。

- 服务层负责调用 Git 工具，不直接散落执行 Git 命令。

- 后续可在该服务中补充同步日志、异常处理和同步状态记录。

### 4. 文档目录入库

- 首次克隆仓库后，对仓库目录进行全量扫描。

- 扫描结果按目录、文件、层级关系等信息保存到数据库。

- 后续执行拉取更新后，检查新增、删除、移动和修改的文件，并同步更新数据库中的目录结构。

### 5. Markdown 内容读取

- 前端查看指定 Markdown 文件时，由后端负责读取原始内容。

- 后端将 Markdown 解析为 HTML，并提取标题结构。

- 前端只负责渲染 HTML 内容和根据标题目录实现页面内跳转。

# 任务模块与 Discord 同步功能

## 目标

- 同步创建：在 OPS 或 Discord 创建任务时，支持在另一个平台自动创建任务；

- 消息互通：在其中一个平台回复消息时可以自动同步到另一个平台；

- 任务跟进：在 OPS 更新任务如状态、分配团队、分配角色等信息时在 Discord 对应帖子推送通知；

## 开发

### 识别关联关系字段

- 任务关联：使用 \`op_task\` 表的 \`url\` 字段保存帖子链接；

- 消息关联：在消息表 \`op_task_reply\` 中新增 \`**externalId**\` 字段，用于保存外部消息的 ID；

- 成员关联：使用角色表 \`base_sys_user\` 中的 \`discordId\` 字段保存对应成员在 Discord 的用户名；

### OPS 创建任务

1. 在 OPS/Discord/Module 模块新增一个 OpsTask 配置，选择连接 Discord 的机器人和该模块可访问的频道；
2. 处理任务创建的方法，根据前端所传的参数分不同情况处理：

   1. 传递 \`{ threadId: 'New' }\` ：在 Discord 创建新帖子；
   2. 传递 \`{ threadId: threadId, isSyncReply: boolean }\` ：根据 \`isSyncReply\` 判断是否读取消息同步到 \`op_task_reply\` 表；
   3. threadId 为空：不做额外处理；

### 消息同步

1. 在 OPS/Discord/Module 模块新增一个 **Public** 配置，作为所有模块一些共用功能使用；
2. 使用 discord.js 包通过 Gateway 方式连接 Discord，OPS 后端可实时接收到新消息并在 \`op_task_reply\` 表同步写入；
3. 在 OPS 发送回复时，使用 Bot 在 Discord 对应帖子发送消息；

### 自定义指令

1. 在 OPS/Discord 模块新增一个 Command 配置模块，用于管理自定义指令的配置；
2. 在添加指令后，将其注册到 OPS/Discord/Module 中配置的 **Public 机器人上；**
3. 在 Gateway 的连接中接收到发送自定义指令事件，并进行相应的处理；

### Discord 创建帖子

1. 当 Gateway 接收到新建帖子时，且所在频道属于 Task 配置范围内，由机器人发送一条带按钮的询问消息；
2. 在用户点击按钮后根据点击的按钮判断是否同步到 OPS，并修改消息将处理结果显示出来；
